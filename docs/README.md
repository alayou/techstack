# Docs

本目录用于沉淀 README 之外的中长期规划和实现拆解。

README 负责说明项目定位、当前能力和快速开始；`docs/` 负责把后续版本目标拆成更具体的产品与工程计划。

## 文档索引

- [路线图](./roadmap.md)：按阶段描述 Techstack 从 MVP 走向 Agent 技术栈治理层的版本方向。
- [开发规划](./development-plan.md)：结合当前代码结构，拆解数据模型、接口、MCP、异步任务和交付顺序。

## 规划原则

- 先把技术栈知识底座做稳，再开放给 Agent 直接使用。
- Agent 可以推荐，但是否采纳必须由开发者决定。
- 收藏、导入、依赖变化和安全数据都应该沉淀为可查询的偏好与治理信号。
- Code Agent 写代码时，不应自由选择技术栈，而应优先使用开发者认可或偏好的包与仓库。
